# 好友機制、狀態追蹤與個人資訊展示實作計畫

本計畫旨在為 SignalR 聊天室整合好友關係資料庫、即時在線與自訂狀態追蹤，以及個人資訊檔案（Profile）的展示功能。

---

## 實作步驟

- [x] **步驟一：建立 `User` 與 `Friendship` 資料模型**
  * **為什麼需要這個步驟**：在 `Models` 新增 `User`（Email、自我介紹 Bio、狀態消息、頭像、註冊時間）與 `Friendship`（好友關係連結）的資料模型，並在 `ChatDbContext` 註冊，以便於 SQLite 資料庫中儲存好友資訊。

- [x] **步驟二：實作資料庫初始化與測試資料 (Seed Data)**
  * **為什麼需要這個步驟**：在資料庫建立時自動塞入測試帳號（如 `Bob`, `Charlie`, `David`）及與新使用者的預設好友關係，方便使用者一登入即有好友名單可供狀態追蹤與互動。

- [x] **步驟三：設計好友管理與個人檔案 API (結合 HybridCache)**
  * **為什麼需要這個步驟**：
    - 提供 `GET /api/users/{username}` 用以獲取對方基本資訊，並套用 `HybridCache` 以防頻繁點擊造成資料庫負擔。
    - 提供 `GET /api/friends` 用以拉取當前使用者的好友列表。
    - 提供 `POST /api/friends/add` 用以主動新增好友。

- [x] **步驟四：於 `ChatHub` 整合即時狀態通知**
  * **為什麼需要這個步驟**：當使用者登入連線（`OnConnectedAsync`）或斷線（`OnDisconnectedAsync`）時，即時廣播通知所有在線好友「我的狀態變更」；並新增 Hub 方法 `UpdateStatusMessage(string msg)` 供使用者更新個人狀態。

- [x] **步驟五：重構前端 UI（好友清單、狀態綠點、個人資訊 Modal）**
  * **為什麼需要這個步驟**：
    - 左側聯絡人清單改版為「我的好友」，並以綠色/灰色圓點動態標示其在線狀態與顯示自訂狀態訊息。
    - 點擊好友頭像或資訊按鈕時，彈出高質感毛玻璃 Modal 視窗，顯示該好友的完整個人基本資訊（Email, Bio, 註冊時間）。
    - 增設「新增好友」按鈕以觸發好友申請。

- [x] **步驟六：端到端整合測試與驗證**
  * **為什麼需要這個步驟**：刪除舊資料庫以套用新 Schema，啟動多視窗進行 Alice、Bob 等帳號的連線，驗證狀態即時切換、個人資訊 Modal 的資料正確性以及新增好友功能。
